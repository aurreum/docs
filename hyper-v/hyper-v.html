Hyper-V User’s Guide#

Introduction#

Aurreum Data Protection Suite (ADPS) supports the backup and restore of virtual machines running on Windows Servers or bare-metal Hyper-V Servers. It is easy to deploy the Hyper-V agent of ADPS. We recommend installing the Hyper-V agent on the Hyper-V server.

Features#

Features#

Feature

Description

Backup source

Virtual machine (VM)

Backup type

Full backup: Back up all the data on the VMs.
Incremental backup: Back up the data that has changed since the last backup.

Backup target

Standard storage pool, de-duplication storage pool, LAN-Free pool

Backup compression

None, fast

Multiple channels

A positive integer from 1 to 32

Backup schedule

Immediate, one-time, hourly, daily, weekly, yearly

Reconnection time

The default value is 10 minutes.

Restore type

Virtual machine restore, virtual disk restore

Restore location

Original host, other hosts

Restore granularity

Disk or virtual machine

Speed limit

The data transfer speed or disk write and read speed can be limited.

D2C

Data can be backed up directly to object storage.

D2T

Data can be backed up directly to the tape library.

Standard checkpoint

Standard checkpoints (i.e., crash-consistent checkpoints) can be created.

Production checkpoint

Production checkpoints (i.e., application-consistent checkpoints) can be created.

Restore the network adapter

The network adapter of the VM can be restored.

Pool replication

D2D2D, D2D2C, D2D2T

Restore from target pools

Restoring from target storage pools is supported.

Transport mode

Network mode: Transport data over the TCP/IP network.
SAN mode: Read backup data directly over SAN not LAN. This mode requires the backup server to read the backup data directly through SAN.

Start virtual machine after restoring

The restored VMs can be started after the restore job is completed.

Download Hyper-V Agent#

Download Windows Package#

  1. Click Resource -> Install Agent icon to access the pop-up window.

  2. Select Windows and click Download Windows agent to download the package.

Install and Configure Hyper-V Agent#

Install and Configure Windows OS Agent#

  1. Double-click the installation package to open the installation wizard.

  2. Select the Hyper-V component manually. You can add or delete other resources according to your needs. If you select a resource, its backup and restore module will be enabled.

  3. Set the Backup Server Host, Backup Server Port, and Access Key. The Access Key identifies the operator of the agent. You can log in to ADPS and see the account’s Access Key on the Personal Settings -> Settings. If the backup server is a non-tenant version, then it is not required to configure the Access Key.

  4. Set the Destination Folder of the backup server.

  5. After the configuration, follow the wizard to complete the installation.

Register Resource#

  1. Log in to ADPS and click the Register icon to register the Hyper-V resource on the Resource interface.

  2. Select the check box to activate the resource.

  3. Assign the resource to user groups.

  4. Click the Modify icon to assign the data network to the resource.

  5. Click Login. Enter the user name and password.

Job#

Log in to ADPS and go to the Job interface.

  1. View the job detail: You can click a Job Name to view the job’s detailed information.

  2. Start: Select a non-running job and click to start the job.

  3. Stop: Select a running job and click to stop the job.

  4. Modify: Select a non-running job and click to modify the job.

  5. Delete: Select a non-running job and click to delete the job.

  6. Disable: Select a non-running scheduled job and click to disable the scheduled job.

Back Up Hyper-V#

Go to the Backup interface.

The steps to create a backup job are:

  1. Hosts and Resources: Select the Hyper-V host that you want to back up. Click Next.

  2. Backup Source: Select Full Backup or Incremental Backup. Choose virtual machines to be backed up. Click Next.

    Backup type:

    • Full backup: Back up all the data on the VMs.

    • Incremental backup: Back up the data that has changed since the last backup (no matter what type of the last backup is, the data that has changed will be backed up).

    Backup source:

    • Virtual machine: Select the VMs that need to be backed up.

  3. Backup Target: Select a storage pool. Click Next.

  • It supports standard storage pool, de-duplication storage pool, and LAN-Free pool (uses SAN transport mode by default).

  1. Backup Schedule: Select a schedule type according to your needs. Click Next.

  2. Backup Options: Set backup options according to your needs. Click Next.

    Common options:

    • Compression: You can select fast compression.

    • Channels: It refers to the maximum number of virtual disks to be backed up at the same time.

    Advanced options:

    • Reconnection time: The job continues after the abnormal reset occurs in the network within the set time. The default value is 10 minutes and the unit is minute.

    • Resumption buffer size: You can set the resumption buffer size. The default value is 10 MiB.

    • Speed limit: You can set different backup speeds in different time periods.

    • Precondition: The precondition is checked before the job starts. The job execution is aborted when the precondition is invalid.

    • Pre/Post action: The pre action is executed after the job starts and before the resource is backed up or restored. The post action is executed after the resource is backed up or restored.

    • Checkpoint consistency level:

      • Crash consistent: Create application-consistent checkpoints that capture the current state of applications.

      • Application consistent: Use backup technology in the guest operating system to create data-consistent checkpoints that do not contain information about running applications.

      • Automatic: Application Consistent is used by default. If it fails, Crash Consistent will be used instead.

  3. Finish: Enter the job name and click Submit.

Restore Hyper-V#

Go to the Restore interface.

The steps to create a restore job are:

  1. Hosts and Resources: Select the Hyper-V host that needs to be restored. Click Next.

  2. Backup Sets: Select Virtual Machine Restore or Virtual Disk as the restore type. Select the restore source. Click Next.

  3. Restore Target: Select a restore target. Click Next.

  4. Restore Schedule: Select a schedule type according to your needs. Click Next.

  5. Restore Options: Set restore options according to your needs. Click Next.

    Common options:

    • Restore location:

      • Original path: The location of the restored configuration file and disk is the same as the original location, i.e., to overwrite and restore to the original path.

      • New path: New virtual machines or virtual disks will be created. It is required to set the name, configuration, and storage directory of the new virtual machines.

    • Channels: It refers to the number of virtual disks that can be restored at the same time.

    • Start virtual machine after restoring: The virtual machine will be started automatically after the restore job.

    • Restore the network adapter: The network adapter of the VM can be restored.

    Advanced options:

    • Reconnection time: The job continues after the abnormal reset occurs in the network within the set time. The default value is 10 minutes and the unit is minute.

    • Resumption buffer size: You can set the resumption buffer size. The default value is 10 MiB.

    • Speed limit: You can set different backup speeds in different time periods.

    • Precondition: The precondition is checked before the job starts. The job execution is aborted when the precondition is invalid.

    • Pre/Post action: The pre action is executed after the job starts and before the resource is backed up or restored. The post action is executed after the resource is backed up or restored.

  6. Finish: Enter the job name and click Submit.

Restore Virtual Machines#

You can select Virtual Machine Restore to create a VM restore job.

Overwrite and Restore to Original Path#

To overwrite and restore to the original path, select the original host as the restore target and the Original Path as the restore location.

Restore to New Path on Original Host#

Select the original host as the restore target and the New Path as the restore location. Set the new VM name and storage directory.

The steps to set a new path:

  1. Rename the new VM:

    (1) Click the VM name to access the Rename window.

    (2) Rename the VM directly or add a prefix and suffix. Click Submit.

    (3) Confirm the new VM name and click Next.

  2. Set the storage directory:

    The steps to set the directories in a batch are:

    (1) Select all the files and click the Set Storage Directories in batch option.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

    The steps to set one directory are:

    (1) Click the storage directory of a specific disk.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

Restore to New Path on Another Host#

Select another host as the restore target and the New Path as the restore location.

Set the new VM name and storage directory.

The steps to create a new path are:

  1. Rename the new VM:

    (1) Click the VM name to access the Rename window.

    (2) Rename the VM directly or add a prefix and suffix. Click Submit.

    (3) Confirm the new VM name and click Next.

  2. Set the storage directory:

    The steps to set the directories in a batch are:

    (1) Select all the files and click the Set Storage Directories in batch option.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

    The steps to set one directory are:

    (1) Click the storage directory of a specific disk.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

Note:

  • Restore to a new path on another host: To restore between hosts with different versions, ADPS only supports restoring from a low version to a high version.

Restore Virtual Disks#

You can select Virtual Disk to create a virtual disk restore job.

Overwrite and Restore to Original Disk#

To overwrite and restore to the original disk, select the original host as the restore target and the Original Path as the restore location.

Restore to New Disk on Original VM#

Select the original host as the restore target and the New Path as the restore location. Set the storage directory of the new disk.

The steps to create a new path are:

  1. Set the storage directory:

    (1) Select all the files and click the Set Storage Directories in batch option, or click the storage directory of a specific disk.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

Select another host as the restore target and the New Path as the restore location.

Then, set the storage directory of the new disk.

The steps to create a new path are:

  1. Set the storage directory:

    (1) Select all the files and click the Set Storage Directories in batch option, or click the storage directory of a specific disk.

    (2) Select the storage directory and click OK.

    (3) Confirm the directory and click Next.

Limitations#

Limitations#

Function

Limitations

Backup

Differential backup is not supported.
VM grouping is not supported.
If the VM to be backed up is overwritten and restored, the backup jobs will be unavailable and should be recreated.
Under the following conditions, the backup job cannot be stopped: The agent is creating checkpoints, or the agent is using a LAN-Free storage pool and mounting the disks.
Checkpoints include crash-consistent checkpoints and application-consistent checkpoints. The default checkpoint is application-consistent.
On Windows Server 2016 and later versions, Hyper-V backup jobs will not reserve checkpoints after the job completion.
A host can only run one backup or restore job at a time. A backup job can back up multiple VMs in parallel, while a restore job can only restore VMs one by one.

Restore

Mount recovery is not supported.
The interface cannot display the restore progress of a single virtual disk.
To overwrite and restore the disk, ensure that the original host is shut down and no checkpoints remain.
To restore between hosts with different versions, ADPS only supports restoring from a low version to a high version.
When the agent is merging subdisks, the restore job cannot be stopped.
ADPS cannot restore the Ubuntu/CentOS Generation 2 virtual machines on Hyper-V 2012 series.
ADPS cannot restore the virtual disks on Ubuntu/CentOS Generation 2 virtual machines.
The Generation 1 virtual machines can only be booted from the IDE controller. The Generation 2 virtual machines can only be booted from the SCSI controller.
